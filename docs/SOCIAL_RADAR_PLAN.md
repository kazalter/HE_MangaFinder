# 作者动态雷达：设计与实施说明

## 目标

为已订阅作者绑定一个或多个本人/社团 X 账号，增量采集本人动态，识别新刊、样张、
封面、参展、预售、再版、延期与取消。高置信情报生成“待发布作品”并通知，但社交动态
永远不会直接写入正式漫画来源或触发下载。

## 模块边界

- `social-collector` 是隔离的 Playwright 容器，只读取 X 页面并返回标准化帖子。登录会话
  位于独立 volume，主应用、Agent 和 QQ Bot 都不能读取 Cookie。
- `app.modules.social` 保存账号、原帖、Agent 审计、情报与通知 outbox，并复用现有持久任务队列。
- 硬规则先排除普通转推并区分参展、再版、取消；DeepSeek 只分析规则选出的候选。
- 本地 Tesseract 处理日文横排/竖排图片，默认只把 OCR 文本发给模型。
- 站内雷达是事实源；QQ 官方 Bot 只发送无封面的纯文字通知与站内回链。

## 判定边界

- 只有 `新刊/新作` 等明确新作信号，加上标题、封面、样张、商品页、展会或摊位中的
  至少两项独立证据，并达到 `0.92`，才允许自动确认。
- `C108` 或摊位号只能证明参展，不能证明存在新作。
- `既刊/再販/再録/重版` 归为旧刊或再版；延期、取消与纠正文案覆盖正面证据。
- Agent 证据必须能回溯到原文、OCR 或结构化事实。Agent 连接失败时降级为待人工确认，
  不允许在已配置 Agent 的情况下靠规则强行自动确认。
- 同一线程、标题、展会与商品编号形成一个情报簇；不同编号或标题不会仅因共同主题而合并。

## 验收重点

- 转推不产生候选；同一帖子重复扫描不产生重复 Agent 审计或 QQ 通知。
- 登录失效、验证码、页面改版、OCR/模型/QQ 故障均失败关闭，并在账号状态中展示原因。
- 人工确认、驳回、关联作品均可审计；后续来源扫描发现正式作品后才进入作品聚合。
- 高置信新作精确率目标不低于 95%，人工候选召回率目标不低于 90%。
